## Final Target

Formalize the all-dimensional odd-modulus theorem:

```lean
theorem odd_modulus_tori_all_dimensions
    {d m : Nat} (hd2 : 2 <= d)
    (hmodd : Odd m) (hm3 : 3 <= m) :
    Shared.CayleyHamiltonDecomposition d m
```

## Main v4 Reset

The shortest proof spine is no longer the direct odd-core split by
`m >= d` and `m < d`.

The v4 manuscript reframes the proof as a dimension-closure theorem:

```text
D2, D3, D5, D7 seeds
+ product closure
+ successor closure b >= 5 solved => 2*b + 1 solved
-------------------------------------------------------
all d >= 2 and all odd m >= 3
```

Thus the old `d < 29` finite boundary table, the `169` boundary pairs, and the
seed-semigroup interval search are not theorem-level inputs.  They remain
audit and regression evidence.

## Preferred Lean Skeleton

Add a narrow successor-closure interface:

```lean
def OddSuccessorClosureGoal : Prop :=
  forall {b m : Nat},
    5 <= b ->
    Odd m -> 3 <= m ->
    StandardCayleySolved b m ->
    StandardCayleySolved (2*b + 1) m
```

Then prove the dispatcher:

```lean
theorem odd_modulus_tori_all_dimensions_of_357_and_successor
    (hSucc : OddSuccessorClosureGoal)
    {d m : Nat} (hd2 : 2 <= d)
    (hmodd : Odd m) (hm3 : 3 <= m) :
    Shared.CayleyHamiltonDecomposition d m
```

Proof shape:

- `2,3,5,7` are imported seeds.
- `4,6,8,9,10` are generated by product closure and induction.
- If `d` is even and `d >= 4`, write `d = 2*(d/2)` and use product closure.
- If `d` is odd and `d >= 11`, write `d = 2*b + 1`; then `b >= 5` and
  `b < d`, so induction gives `D_b(m)`, and `hSucc` gives `D_d(m)`.

This should replace the older top-level reliance on a general
`OddCoreSmallModulusSlackPacketLiftGoal` dispatcher.  The general small-branch
interface remains useful, but it is no longer the cleanest top-level route.

## How to Prove the Successor Closure

The successor closure itself splits by modulus size:

```text
target d = 2*b + 1

if m >= d:
  use the count branch

if m < d:
  use base-tail Hall-slack with
    T = d - b = b + 1
    packets 3,2,2,...,2
```

The packet condition is:

```text
m = 1 + (m - 1)          for k = 2
m = 1 + 1 + (m - 2)      for k = 3
```

all parts being positive units modulo odd `m`.

The slack check reduces to:

```text
3^b > 3*(2*b + 1)*(b + 1)
```

at `b = 5`, then follows by monotonic growth.

## Remaining Lean Blocks

The active proof obligations should now be phrased as four construction
families plus one small dispatcher theorem:

1. Count branch for `m >= d`.

   Formalize the v4 symbolic constructions:

   ```lean
   OddCoreHighModulusPrefixCountGoal
   ```

   Internally this still decomposes into:

   ```lean
   PrefixCount.MarginTransportQge2CompatibleGoal
   PrefixCount.MarginTransportQeq1CompatibleGoal
   PrefixCountRootFlatCanonicalReturnGoal
   ```

   The paper-level q>=2 and q=1 constructions are now explicit in v4; the Lean
   work is to formalize them, not to search for new mathematics.

2. Root-flat canonical return.

   Prove that a prefix-admissible count matrix yields the canonical root-flat
   certificate with

   ```lean
   cert.schedule.step = prefixCountRootStep d m
   ```

3. Base-tail Hall-slack successor theorem.

   The narrow top-level target is successor closure, but it should be proved
   through the reusable Hall-slack machinery.  Existing broader endpoint:

   ```lean
   OddCoreSmallModulusSlackPacketLiftGoal
   ```

   remains a valid sufficient route, but a narrower
   `OddSuccessorSmallModulusGoal` may be faster to close.

4. Active Hall realization.

   Formalize the v4 sequence:

   ```text
   Active Hall criterion
   mixed-vertex slack
   controlled residue rounding
   active Hall-slack realization
   ```

   Current Lean support in `RoundComposite/ActiveHall.lean` covers many cut and
   compatibility lemmas but not the full Hoffman/rounding realization theorem.

5. Closure dispatcher.

   Prove the strong-induction theorem from D2/D3/D5/D7, product closure, and
   `OddSuccessorClosureGoal`.  This is expected to be small compared with the
   construction blocks.

## Deprioritized From the Main Goal

- The `d < 29` finite boundary witness table.
- The former special status of `(d, m) = (13, 3)`.
- The direct seed-semigroup small branch as the top-level all-dimensional
  dispatcher.
- The obsolete q=1 plus-family global target, which Lean already proves is
  impossible.

## Status Distinction

At paper level, v4 presents proofs of the count branch and the Hall-slack
base-tail branch.

At Lean level, these remain construction endpoints to formalize.  The final
Lean theorem is still absent until the successor closure and its count/base-tail
ingredients are closed without assumptions.

## One-Sentence Goal

Formalize the v4 closure theorem: D2/D3/D5/D7 seeds plus product closure and
the successor closure `b -> 2*b+1` generate Hamilton decompositions for every
dimension `d >= 2` and every odd modulus `m >= 3`.
